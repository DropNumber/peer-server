Peer-to-Peer Server with WebRTC

Sophia, Brie, and Joe

Spring 2013, Stanford Senior Project

About

We have implemented a peer-to-peer client server using WebRTC, where one browser can act as a server for other browsers across WebRTC peer-to-peer data channels. Anyone can create a client-server within their browser tab, upload content, and even generate dynamic results. Any client browser that connects to that client server will behave as though the client server were a "real" server serving content.

The real server only connects the client-browsers with the client-server; from there, the connection will continue to work even if the real server goes down. Furthermore, this system allows for people to create decentralized web sites where all the content on the server lives within their browser, not on any external server.
